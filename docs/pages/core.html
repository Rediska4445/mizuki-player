Mizuka Core

Главным компонентом является само приложение, которое подключает другие свои внутренние модули.

Внешняя структура: зависимости и ресурсы

С внешней точки зрения приложение использует сторонние файлы для работы, а именно:

Библиотеки (libraries/)

Разделяются на 3 типа для чёткой структуры:

  • classes/ — содержит библиотеки, которые подключаются через классовую систему.
  • modules/ — содержит библиотеки, которые подключаются через модульную систему.
  • bin/ — содержит бинарные файлы (в том числе JavaFX).

Ресурсы (res/)

Ресурсы определяются приложением через файл resources.properties (формат свойств с использованием переменных).

  • audio — аудиофайлы
  • lang — локализация
  • mods — моды (необязательно)
  • styles — стили
  • visual — UI-элементы

Кэш (cache/)

Содержит данные, сохранённые плеером:

  • Локальный (cache/):
    • плейлисты (playlists/)
    • теги (tags/)
    • треки (tracks/)
  • Сетевой (inet/)
  • Общий кэш (shared/)

Конфигурация (config/)

Содержит файлы для внешней конфигурации приложения:

  • горячие клавиши (bindings/)
  • логирование (log/)
  • файл конфигурации settings.cfg

Лицензии (license/)

Содержит файлы лицензий использованных в приложении инструментов.

Внутренняя структура: модули

С внутренней точки зрения приложение использует следующие модули:

  • Spotify API Wrapper — интеграция со Spotify (подр. см. соот. документацию)
  • VST-3 Host — поддержка VST-плагинов (подр. см. соот. документацию)
  • DWM API — интеграция с Desktop Window Manager
  • Mizuka Std Utils — стандартные утилиты (подр. см. соот. документацию)
  • TempoShifter — изменение темпа аудио
